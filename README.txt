Были выполнены все основные пункты, кроме 3 в API для администратора в силу нехватки времени.
Примеры REST-запросов:
API ДЛЯ ПОЛЬЗОВАТЕЛЯ
1. Регистрация нового пользователя:

POST запрос
url:http://localhost:8080/signUp
Происходит генерация ключа на основе полученных данных и добавление сведений в БД. Если пользователь уже зарегистрирован, то вернется ошибка: "В регистрации отказано, данный пользователь уже существует."
Запрос:
{
    "username": "petyarulit3000",
    "email": "petrcool@mail.ru"
}
Ответ:
{
    "secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda"
}
2. Просмотр баланса своего кошелька:

GET запрос
url:http://localhost:8080/balance
На вход передается ключ пользователя, а затем по нему в БД осуществляется поиск кошелька данного пользователя. В случае неправильного ключа возвращается ошибка: "Пользователя с данным ключом не существует."
Запрос:
{
	"secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda"
}
Ответ:
{
    "TON_wallet": 6.00,
    "BTC_wallet": 5.00000000,
    "RUB_wallet": 5000.00
}
3. Пополнение кошелька:

POST запрос
url:http://localhost:8080/replenishment
На основе ключа, происходит поиск кошелька пользователя, а затем происходит проверка, какой счет необходимо пополнить. Учитывается проверка пополняемой суммы, чтобы она была положительной.
В случае ввода некорректной суммы вернется ошибка: "Введенное значение баланса некорректно."
Запрос:
{
	"secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda",
	"RUB_wallet": "1000"
}
Ответ:
{
    "RUB_wallet": "6000.00"
}
4. Вывод денег с биржи:

POST запрос
url:http://localhost:8080/cashout
На основе ключа, происходит поиск кошелька пользователя, затем проверяется корректность выводимой суммы, чтобы она не превышала исходный баланс.
В случае некорректного значения вернется ошибка: "Недостаточно средств на балансе."
Запрос:
{
	"secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda",
	"currency": "RUB",
	"count": "100",
	"credit_card": "1234 5678 9012 3456"
}
Ответ:
{
    "RUB_wallet": "5900.00"
}
5. Просмотр актуальных курсов валют:

GET запрос
url:http://localhost:8080/rate
На вход передается ключ и значение валюты, курс которой необходимо просмотреть. Данный метод доступен для пользователя и администратора.
Запрос:
Пользователь:
{
	"secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda",
	"currency": "TON"
}
Ответ:
{
    "RUB": "185",
    "BTC": "0.000055"
}
Администратор:
{
	"secret_key": "secretadminkey",
	"currency": "TON"
}
Ответ:
{
    "RUB": "185",
    "BTC": "0.000055"
}
6. Обмен валют по установленному курсу:

POST запрос
url:http://localhost:8080/trade
На основе ключа, происходит поиск кошелька пользователя, затем проверяется баланс валюты, которую необходимо обменять. В случае, когда значение вывода превышает баланс возвращается ошибка:
"Недостаточно средств на балансе для обмена." Затем происходит обмен валюты по установленному курсу, который хранится в БД, и пополнение соответствующего кошелька на данное значение.
Запрос:
{
	"secret_key": "980a214b4f8ef6b9486f63b63f86321d650232864c89710b530beffadb784dda",
	"currency_from": "RUB",
	"currency_to": "TON",
	"amount": "10"
}
Ответ:
{
    "currency_from": "RUB",
    "currency_to": "TON",
    "amount_from": "10",
    "amount_to": "1800"
}
API ДЛЯ АДМИНИСТРАТОРА
1. Изменить курс валют:
POST запрос
url:http://localhost:8080/updateExchange 
На основе ключа администратора, который хранится в БД, а не выдается при регистрации, происходит изменение значений в таблице курса валют.
Запрос:
{
    "secret_key": "secretkeyadmin",
    "base_currency": "TON",
    "BTC": "0.000056",
    "RUB": "184"
}
Ответ:
{
    "RUB": "184",
    "BTC": "0.000056"
}
2. Посмотреть общую сумму на всех пользовательских счетах для указанной валюты:
GET запрос
url:http://localhost:8080/getAll
На основе ключа администратора, который хранится в БД, а не выдается при регистрации, происходит получение информации о количестве денежных средств всех пользователей
в необходимой валюте.
Запрос:
{
	"secret_key": "secretkeyadmin",
	"currency": "RUB"
}
Ответ:
{
    "RUB": "28190.00"
} 
Приложение подключено к БД MySql, в которой осуществляется хранение данных пользовательских кошельков, ключей, ролей и курса валют. Все запросы проверяются на корректность, в случае некорректного запроса 
возвращается ошибка 400 BAD_REQUEST. 
Если пользователь пытается произвести операции администратора, возвращается ошибка 403 Forbidden - доступ запрещен.
